# B2B AI 에이전트 플랫폼 소개 및 문의 관리 시스템 MVP PRD

## 🎯 핵심 정보

**목적**: B2B 고객에게 AI 에이전트 오케스트레이션 시스템 및 3가지 전문 에이전트를 소개하고, 웹 문의를 노션으로 자동 전송하여 효율적으로 관리
**사용자**: AI 에이전트 도입을 검토하는 B2B 기업 담당자 및 의사결정권자
**디자인 소스**: 피그마 (모든 6개 페이지 디자인 완성)

## 🚶 사용자 여정

```
1. 메인 페이지 (서비스 전체 개요)
   ↓ [각 에이전트 카드 클릭 또는 네비게이션]

2. Orchestration System 소개
   ↓
   리더 에이전트 소개
   ↓
   캐릭터챗 에이전트 소개
   ↓
   프로프 리팅 에이전트 소개
   ↓ [문의하기 버튼 클릭]

3. 문의 폼 페이지
   ↓ [폼 작성 및 제출]

   [검증 성공] → 노션 API 전송 → 제출 완료 상태
   [검증 실패] → 에러 표시 및 수정 요청
   ↓

4. 제출 완료 상태
   ↓ [사용자 선택]

   - 홈으로 돌아가기 → 메인 페이지
   - 추가 문의하기 → 문의 폼 페이지 (초기화)
```

## 📝 개발 시 주의사항

### 1. 피그마 디자인 충실도

- 피그마 디자인에 모든 페이지가 완성되어 있으므로, 디자인을 최대한 충실하게 구현
- 피그마 MCP를 활용하여 색상, 타이포그래피, 간격, 레이아웃을 정확히 추출
- 각 에이전트 설명 콘텐츠도 피그마에서 추출하여 사용

### 2. Server Actions 활용

- 폼 제출은 Server Actions를 사용하여 서버 측에서 안전하게 처리
- 클라이언트에서 노션 API 키 노출 방지

### 3. 환경 변수 보호

- Notion API 키는 절대 클라이언트에 노출하지 않음 (NOTION_API_KEY는 서버 전용)
- .env.local 파일을 .gitignore에 추가하여 버전 관리에서 제외

### 4. Zod 스키마 재사용

- 클라이언트 검증과 서버 검증에서 동일한 Zod 스키마 사용
- 타입 안전성 보장 및 중복 코드 방지

### 5. 에러 경계

- Notion API 실패 시에도 사용자에게 명확한 피드백 제공
- 네트워크 오류, 서버 오류 등 다양한 에러 케이스 처리

### 6. 접근성

- 폼 요소에 적절한 label, aria-label 사용
- 키보드 네비게이션 지원 (Tab, Enter, Escape)
- 색상 대비 (WCAG AA 기준)

### 7. SEO 최적화

- 각 페이지마다 고유한 metadata (title, description) 설정
- Open Graph 태그로 소셜 미디어 공유 최적화
- 의미 있는 HTML5 시맨틱 태그 사용

### 8. 성능 최적화

- Next.js 16의 Turbopack 활용 (빠른 빌드)
- 이미지 최적화 (next/image 사용)
- 코드 스플리팅 (페이지별 자동 분할)
- 동적 임포트 (필요시)

- 모바일 우선 접근 (Mobile First)
